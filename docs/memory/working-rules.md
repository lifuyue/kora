# Working Rules

## 适用范围
这里只记录稳定、长期有效、会直接影响实现决策的项目规则。不要写任务过程，不要写一次性讨论结论。

## 当前规则
### 快速迭代优先
项目处于快速迭代阶段，优先选择收敛复杂度、提升可维护性的方案，而不是保留历史结构。

### 允许激进重构
当现有实现阻碍继续推进时，可以直接做模块拆分、命名调整、状态流重组和实现替换，不必为了“看起来改动小”而保留坏结构。

### 默认不兼容旧数据
除非需求明确要求兼容，否则不为旧数据库、旧缓存、旧偏好配置或过渡数据格式设计兼容层和迁移链路。

### 优先删除复杂兼容层
面对临时补丁、双写、分支兜底或历史兼容胶水代码时，优先删除和重建，避免继续叠加。
